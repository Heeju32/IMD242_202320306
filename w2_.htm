좌표계

컴퓨터 그래픽스에서 화면 위의 어느 위치를 표현하는데는 데카르트 좌표계 (Cartesian Coordinate System)을 사용
Cartesian Coordinate System에 대한 더 자세한 정보는 아래 링크들을 참조
데카르트 좌표계
데카르트 좌표계(영어: Cartesian coordinate system)는 임의의 차원의 유클리드 공간(혹은 좀 더 일반적으로 내적 공간)을 나타내는 좌표계 중 하나이다. 천장을 날아다니며 옮겨붙는 파리를 통해 영감을 얻어 해당 좌표계를 발명한 프랑스의 철학자이자 수학자인 르네 데카르트의 이름을 따서 지어졌다. 2차원 데카르트 좌표계는 좌표평면(座標平面, 영어: coordinate plane), 3차원 데카르트 좌표계는 좌표공간(座標空間, 영어: coordinate space)이라고도 한다. 직교 좌표계(直交座標系, 영어: orthogonal coordinate system)는 데카르트 좌표계를 포함하여 극좌표계, 원통좌표계, 구면좌표계 등 좌표축과 평행한 단위벡터끼리 항상 서로 수직한 모든 좌표계를 총칭하는 표현이다. 데카르트 좌표계는 극좌표계 등 다른 좌표계와 달리, 임의의 차원으로 쉽게 일반화할 수 있다. 데카르트 좌표계는 나타내는 대상이 평행 이동에 대한 대칭을 가질 때 유용하나, 회전 대칭 등 다른 꼴의 대칭은 쉽게 나타내지 못한다. 일반적으로 주어진 유클리드 공간에 기저와 원점이 주어지면 이를 이용하여 데카르트 좌표계를 정의할 수 있다.
https://ko.wikipedia.org/wiki/데카르트_좌표계
namu.wiki
https://namu.wiki/w/좌표계#s-2.1
다른 좌표계로 극좌표계 (Polar Coordinate System)이 있다. 더 자세한 정보는 아래 링크들을 참조
극좌표계
극좌표계(極座標系, 영어: polar coordinate system)는 평면 위의 위치를 각도와 거리를 써서 나타내는 2차원 좌표계이다. 극좌표계는 두 점 사이의 관계가 각이나 거리로 쉽게 표현되는 경우에 가장 유용하다. 데카르트 좌표계에서는 삼각함수로 복잡하게 나타나는 관계가 극좌표계에서는 간단하게 표현되는 경우가 많다. 2차원 좌표계이기 때문에 극좌표는 반지름 성분과 각 성분의 두 성분으로 결정되며 주로 로 나타내는 반지름 성분은 극(데카르트 좌표에서 원점)에서의 거리를 나타낸다. 주로 로 나타내는 각 성분은 0°(데카르트 좌표계에서 x축의 양의 방향에 해당)에서 반시계 방향으로 잰 각의 크기를 나타낸다.
https://ko.wikipedia.org/wiki/극좌표계
namu.wiki
https://namu.wiki/w/좌표계#s-2.2
일반적인 데카르트 좌표계의 원점 (0, 0)은 좌측 하단 (음수 개념이 없는 경우), 혹은 중앙 (음수 개념이 있는 경우)에 위치하지만, 컴퓨터 그래픽스에서 원점은 좌측 상단에 위치
일반적인 데카르트 좌표계의 수직축 (보통 y축)은 위로 갈수록 값이 증가하지만, 컴퓨터 그래픽스에서는 아래로 갈수록 값이 증가
위와 같은 이유로 일반적인 데카르트 좌표계에서 각도는 반시계방향으로 커지지만, 컴퓨터 그래픽스에서는 시계방향으로 커진다.

Function Call

함수를 실행하려면 아래와 같은 형식에 따라 명령문을 작성
함수이름(인자1, 인자2, …);
함수이름은 띄어쓰기를 사용하지 않고 띄어쓰기가 필요한 경우 대문자를 사용
hello world → helloWorld
이는 띄어쓰기가 허용되지 않는 상태에서 띄어쓰기를 나타내는 여러가지 방법 중 하나로 낙타의 등과 같다고 해서 camelCase라 부름
다른 방법들로는 다음과 같은 것들이 있음 (다른 프로그래밍 언어에서는 활용되나 JAVA 및 processing에서는 camelCase를 사용)
snake_case
kebab-case
열고 닫는 괄호는 필수
세미콜론 또한 필수
인자는 함수에 따라 하나도 요구하지 않을 수도 있고, 하나만 요구하거나, 여러개를 요구
fullscreen();
size(640, 480);
circle(320, 240, 100);
여러 인자를 함수에 넣을 때는 각 인자 사이에 쉼표를 넣어 구분. 띄어쓰기는 필수는 아니지만 가독성을 위해 넣는 것을 추천

The Processing Reference

Processing의 함수를 어떻게 사용하는지 확인하고 싶다면 함수이름 위에서 우클릭 → Find in reference를 누른다.
Processing의 모든 함수 (및 구조)는 아래 링크에서 확인할 수 있다.
Reference
Find further documentation of the Processing language
https://processing.org/reference
각 페이지는 다음의 정보를 담고 있다.
Description: 해당 함수가 어떤 역할을 하는지 설명한다.
Examples: 해당 함수의 다양한 사용 예시를 통해 해당 함수의 사용방법을 알려준다.
Syntax: 몇 개의 인자가 입력되어야 하는지 알 수 있다.
Parameters: 각 매개변수가 무엇을 의미하는지 알 수 있다.
(Type): 매개변수의 데이터 타입이 무엇인지 알 수 있다.
Return: 함수가 어떤 데이터 타입을 반환하는지 알 수 있다.
Related: 함께 사용되는 함수들의 목록이다.
우선은 2D Primitives를 살펴보라.

Reference Point

Java
복사
size(640, 480); background(255); circle(320, 240, 100); square(480, 120, 100);
​
정사각형이 원에서 예상되었던 위치와는 다른 위치에 그려질 것이다. 왜냐하면 기준점 (Referece Point)가 다르기 때문이다.
rect(), square()의 기준점을 변경하려면 rectMode()를 사용한다.
생각보다 많은 방식이 존재하지만 CENTER와 CORNER만 알아도 충분하다.
